Проєкт: Stardew Valley – Save / Load System QA

Цей документ описує план тестування для модуля Save / Load System у грі Stardew Valley (PC, Steam).
Метою є перевірка коректності створення, завантаження та збереження ігрових сейвів, а також стабільності системи під час нестандартних ситуацій.

2. Обсяг тестування (Scope)

In Scope (що тестуємо):

Створення нового сейву

Автозбереження наприкінці ігрового дня

Завантаження існуючих сейвів

Повторний запуск гри та збереження прогресу

Робота з декількома сейвами

Обробка аварійного виходу під час збереження

Out of Scope (що НЕ тестуємо):

Ігрові механіки (бойові дії, квести, фермерські процеси)

Моди та сторонні додатки

Баланс гри та продуктивність

3. Типи тестування

Функціональне тестування (Functional Testing): перевірка, чи працює система Save / Load за очікуванням

Ручне тестування (Manual Testing): всі перевірки виконуються вручну у грі

Тестування на крайніх випадках (Edge Case Testing): закриття гри під час збереження, пошкоджені сейви, декілька сейвів

4. Тестове середовище (Test Environment)

Платформа: PC

ОС: Windows 11

Версія гри: Steam (остання стабільна)

Введення: клавіатура та миша

Інструменти: GitHub для документування, Markdown файли для чеклістів та баг-репортів

5. Критерії входу та виходу (Entry / Exit Criteria)

Entry Criteria (умови початку тестування):

Гра встановлена та запущена на PC

Наявність нового або існуючого сейву

Структура репозиторію GitHub підготовлена (чеклісти, тест-кейси, баг-репорти)

Exit Criteria (умови завершення тестування):

Всі тест-кейси пройдені або документовані як Failed / Blocked

Усі знайдені баги описані в баг-репортах

Підготовлений Test Run Report з підсумками

6. Ризики (Risks)

Аварійне завершення гри може пошкодити сейви

Проблеми з Steam версією гри або оновлення можуть вплинути на тестування

Обмежена кількість тестових сценаріїв може не покрити всі edge cases

7. Вихідні матеріали тестування (Test Deliverables)

8. Підсумок

Цей план тестування визначає системний підхід до перевірки модуля Save / Load у Stardew Valley.
Він слугуватиме основою для руху від чеклістів до тест-кейсів, баг-репортів та підсумкового звіту.
